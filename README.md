# Calculator (Vanilla JS)

Простой калькулятор на чистом JavaScript с вводом через кнопки и дисплеем.

## Функциональность
- ввод чисел (многозначные)
- десятичные числа через точку `.`
- операции: `+`, `-`, `*`, `/`
- вычисление по кнопке `=`
- цепочка операций слева направо (например: `2 + 3 + 4 =`)
- замена оператора при повторном нажатии (например: `2 + * 3 =` -> считается как `2 * 3`)
- кнопка `C` полностью сбрасывает состояние
- деление на ноль: показывает `Error`
- после `Error` кнопки операторов и `=` игнорируются, пока не введено число или не нажата `C`

## Как устроена логика
Калькулятор работает через состояние (state). Дисплей (`input#display`) только отображает текущее значение.

Используются 4 переменные состояния:
- `displayValue` - строка, что показывается на дисплее (например `"0"`, `"12"`, `"0."`)
- `firstValue` - первое число / промежуточный результат (number или `null`)
- `operator` - текущая операция (`"+", "-", "*", "/"` или `null`)
- `waitingForSecondValue` - флаг ожидания ввода второго числа (boolean)

## Обработка кликов
На контейнер кнопок `#keys` повешен один обработчик `click`.
У каждой кнопки есть `data-action`:
- `number` - цифры
- `decimal` - точка
- `operator` - операции
- `equals` - равно
- `clear` - очистка

## Особые случаи
### Деление на ноль
При `operator === "/"` и `secondValue === 0`:
- на дисплей выводится `Error`
- состояние сбрасывается (`firstValue = null`, `operator = null`)
- `waitingForSecondValue = true`, чтобы следующий ввод числа начинал новое значение

### Защита после `Error`
Если на дисплее `Error`, то:
- `operator` и `equals` игнорируются
- `number`, `decimal`, `clear` работают и позволяют продолжить
